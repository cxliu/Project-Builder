Known Issues In M-Build


M-Build Core

MB-46: There is no way to set compile flags per file. In the future, this will be possible through the tag syntax.

ARM

MB-92: Linker fails with error when building dynamic libraries on ARM.

Visual Studio

MB-89: Visual Studio 2010 has a known issue with batch building. See Batch Build links to wrong referenced projects for more details. This bug will be triggered if you do a batch rebuild of some of the projects in example-2. It is possible that in a future release of M-Build, the workaround given in the above URL will be used; however, it is not clear if it will be suitable yet.

Makefiles

MB-5: The language used for specifying the makefile templates is not great for extensibility. If you need to make changes to the generated Makefiles, you may be a little bit stuck depending on how significant the changes are.
